# CLI reference

`shelf` is a single static binary. Build it with `make cli` (or both binaries
with `make build`). Run any command with `--help` for its flags.

Valid service names: `postgres`, `redis`, `rabbitmq`, `mongodb`, `aistor` (S3,
requires `make s3-up`), `signoz` (observability, requires `make signoz-up`).

## Provisioning

### `shelf setup <app> -s <services>`
Provision resources for a new app. Each service gets an isolated user/database
and a generated password; prints a ready-to-paste `.env` block.

| Flag | Description |
| --- | --- |
| `-s, --services` | Comma-separated services, e.g. `postgres,redis,rabbitmq,mongodb`. |
| `--env <name>` | Tag a single app with this environment (used for the SignOz `service.name`). |
| `--envs <csv>` | Expand into sibling apps — `--envs staging,prod` creates `<app>-staging` and `<app>-prod` sharing one SignOz service name. Mutually exclusive with `--env`. |
| `--full-access` | Grant full data-plane access (no key/db isolation). Use sparingly. |

```bash
shelf setup myapp -s postgres,redis,rabbitmq,mongodb
shelf setup api --envs staging,prod -s postgres,signoz
```

### `shelf add <app> -s <services>`
Attach more services to an existing app.

```bash
shelf add myapp -s signoz
```

### `shelf detach <app> -s <services>`
Detach an addon (registry-only, no teardown). Only addons without per-app
credentials are detachable — currently `signoz`.

## Inspecting

### `shelf list [--json]`
List all provisioned apps. `--json` emits the raw registry for scripting.

### `shelf credentials <app>`
Reprint the `.env` block (connection strings) for an app.

### `shelf status`
Show infrastructure container status (`docker compose ps`).

## Removing

### `shelf remove <app> [-s <services>] [--force]`
Remove an app's resources. With `-s`, remove only specific services; otherwise
the whole app. `--force` skips confirmation.

## Backups

See [BACKUPS.md](BACKUPS.md) for the full workflow.

### `shelf backup [<app>] [--all] [-s <services>]`
Back up one app, or `--all` apps. `-s` limits to specific services.

### `shelf backup delete <app> <file>`
Delete a specific backup file (path traversal is rejected).

### `shelf restore <app> [--file <path>] [-s <services>] [--force]`
Restore from the latest backup, or an explicit `--file`. The service is inferred
from the filename prefix when restoring a single file.

```bash
shelf backup myapp
shelf backup --all
shelf restore myapp --file backups/myapp/postgres_20260405T0300.sql
```

## Scheduled backups

Schedules are shared with the web UI (stored in the app SQLite database).

### `shelf schedule list`
List backup schedules.

### `shelf schedule create <app> --cron "<expr>" [flags]`

| Flag | Description |
| --- | --- |
| `--cron "<expr>"` | Cron expression (required). |
| `-s, --services` | Limit the schedule to specific services. |
| `-z, --timezone` | IANA timezone for the cron expression. |
| `--retention-days N` | Delete backups older than N days (`0` disables). |
| `--retention-count N` | Keep the last N files per app/service (`0` disables). |
| `--disabled` | Create the schedule paused. |

### `shelf schedule pause|resume|delete <id>`
Manage an existing schedule by id.

## Infrastructure

### `shelf start [docker compose args...]`
Start the infrastructure (`docker compose up -d`). Extra args pass through.

### `shelf reconcile`
Re-apply per-app resources (vhosts, users, permissions, databases) from the
registry against the running containers. Idempotent; runs automatically on every
`docker compose up`. See [Architecture → Reconcile](ARCHITECTURE.md#reconcile).

### `shelf registry encrypt`
Re-encrypt the registry in place using `INFRA_SHELF_SECRET`.

> **Tip:** the connection-string format and env-var names for each service are
> shown in the [README → Setup Output](../README.md#quick-start) and generated by
> `internal/envspec`.
